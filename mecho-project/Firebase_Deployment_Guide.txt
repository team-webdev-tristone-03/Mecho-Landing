MECHO PROJECT - FIREBASE DEPLOYMENT GUIDE
==========================================

This document explains the complete step-by-step process of how the MECHO React project was deployed to Firebase Hosting.

OVERVIEW
--------
The MECHO project is a React web application built with Vite that was successfully deployed to Firebase Hosting. The deployment process involved creating Firebase configuration files, building the project, and deploying to Firebase.

PREREQUISITES
-------------
1. Node.js and npm installed
2. Firebase CLI installed globally (npm install -g firebase-tools)
3. Firebase account with access to create projects
4. React project built with Vite

STEP-BY-STEP DEPLOYMENT PROCESS
===============================

STEP 1: PROJECT STRUCTURE VERIFICATION
--------------------------------------
Before deployment, the project had the following structure:
- React app built with Vite
- Source code in src/ directory
- Build configuration in vite.config.js
- Package.json with build scripts

STEP 2: FIREBASE CONFIGURATION FILES CREATION
---------------------------------------------
Two essential Firebase configuration files were created:

A) firebase.json
Created in project root with the following content:
{
  "hosting": {
    "public": "dist",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}

Purpose: Configures Firebase hosting settings
- "public": "dist" - Points to the build output directory
- "rewrites" - Enables single-page application routing

B) .firebaserc
Created in project root with the following content:
{
  "projects": {
    "default": "mecho-landing"
  }
}

Purpose: Specifies which Firebase project to use for deployment

STEP 3: PROJECT BUILD
--------------------
Command executed: npm run build

This command:
- Runs Vite build process
- Compiles React components
- Optimizes assets (images, CSS, JS)
- Creates production-ready files in dist/ directory

Build Output:
- dist/index.html (5.94 kB)
- dist/assets/index-B2mGecDT.css (85.49 kB)
- dist/assets/index-BUh-__by.js (436.64 kB)
- Various image assets (Clean.jpg, Dusty.jpg, etc.)

STEP 4: FIREBASE PROJECT SELECTION
----------------------------------
Command executed: firebase use mecho-landing

This command:
- Sets the active Firebase project to "mecho-landing"
- Ensures deployment targets the correct project
- Confirms project access and permissions

STEP 5: DEPLOYMENT TO FIREBASE HOSTING
--------------------------------------
Command executed: firebase deploy --only hosting

This command:
- Uploads all files from dist/ directory to Firebase Hosting
- Configures hosting rules and redirects
- Makes the website live on Firebase's CDN

Deployment Process:
1. Found 9 files in dist directory
2. Uploaded new files (4/9 files were new/changed)
3. Finalized version on Firebase servers
4. Released new version to production
5. Made website live at hosting URL

STEP 6: DEPLOYMENT SUCCESS
-------------------------
Final Result:
- Project Console: https://console.firebase.google.com/project/mecho-landing/overview
- Live Website: https://mecho-landing.web.app

FIREBASE PROJECT CONFIGURATION
==============================

The project uses Firebase for two purposes:

1. HOSTING (Deployment)
   - Serves the React application
   - Handles routing for single-page application
   - Provides CDN for fast global access

2. FIRESTORE DATABASE (Contact Forms)
   - Stores contact form submissions
   - Project ID: mecho-landing
   - Collection: ContactForm
   - Integration in Contact.jsx component

Key Success Factors:

2. Proper build output directory (dist/) in firebase.json
3. Successful production build before deployment
4. Firebase CLI authentication and project access
